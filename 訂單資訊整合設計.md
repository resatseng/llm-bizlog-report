## 一、整合目標

在現有的 **Output 3：機會卡（B+A 高溫標的）** 中整合合約與商機歷程資訊，提供業務團隊：

### 1.1 已成交客戶分析
- **合約狀態管理**
  - 有效合約客戶的法人類型分布
  - 合約產品類別偏好
  - 首約 vs 續約客戶比例
  - 合約到期預警（30/60/90 天內到期）

- **續約規律分析**
  - 續約週期（合約期限：年約/季約/月約）
  - 續約率（按法人類型、產品類別）
  - 產品升級路徑（基礎版→專業版→企業版）
  - 續約前的商機階段變化

### 1.2 未成交潛在客戶分析（沉睡客戶喚醒）
- **沉睡客戶定義**
  - 有商機記錄（E/D/C2/C1 階段）
  - 從未成交（無合約記錄）
  - 超過 90 天無互動
  - 未被明確拒絕（無「專案不進行」標記）

- **喚醒策略**
  - 以活動邀約為主（研討會、產品發表會、免費試用）
  - 依法人類型與痛點類型，推薦合適活動主題
  - 追蹤最後接觸日期，優先喚醒「溫度較高」的潛客

### 1.3 商機優先級優化
- 結合合約歷史，提高機會卡的準確度
- 識別高價值客戶：續約客 > 升級客 > 即將到期 > 新客
- 預測續約風險與流失預警

## 四、合約與續約路徑分析設計

### 4.1 合約時間序列分析

**目標：** 追蹤每家公司的合約歷程與續約模式

**實作方法：**
```python
# 為每筆合約標註序號（第幾次合約）
contract_details['contract_seq'] = (
    contract_details.sort_values('contract_start_date')
    .groupby('company_id').cumcount() + 1
)

# 識別續約行為（前一份合約存在）
contract_details['is_renewal'] = contract_details['contract_seq'] > 1

# 計算合約到期天數
contract_details['days_to_expiry'] = (
    pd.to_datetime(contract_details['contract_end_date']) - pd.Timestamp.now()
).dt.days

# 建立合約序列字串
contract_sequences = (
    contract_details.sort_values(['company_id', 'contract_start_date'])
    .groupby('company_id')
    .apply(lambda g: ' → '.join(
        f"{row['product_name']}({row['contract_duration_months']}月)"
        for _, row in g.iterrows()
    ))
    .reset_index(name='contract_sequence')
)
```

**輸出範例：**
```
company_id | contract_sequence
---------- | -----------------------------------------------
GY001-001  | ERP基礎版(12月) → ERP專業版(12月) → ERP企業版(24月)
GY001-002  | CRM標準版(12月) → CRM標準版(12月)
```

### 4.2 續約時產品升級路徑追蹤

**目標：** 識別續約時的產品版本升級行為

**產品版本映射表（需手動建立）：**
```python
PRODUCT_HIERARCHY = {
    "ERP系統": ["基礎版", "專業版", "企業版"],
    "CRM系統": ["入門版", "標準版", "旗艦版"],
    "資安產品": ["單點防護", "整合防護", "全方位防護"]
}
```

**升級識別邏輯：**
```python
def detect_renewal_upgrade(df):
    """識別續約時的產品升級"""
    upgrades = []
    for cid, group in df.groupby('company_id'):
        group = group.sort_values('contract_start_date')
        
        for product_line, versions in PRODUCT_HIERARCHY.items():
            contracts = group[group['product_name'].str.contains(product_line)]
            
            # 至少有兩份合約才能判斷升級
            if len(contracts) >= 2:
                for i in range(len(contracts) - 1):
                    c1 = contracts.iloc[i]
                    c2 = contracts.iloc[i+1]
                    
                    # 判斷版本是否升級
                    try:
                        v1_idx = versions.index(c1['product_version'])
                        v2_idx = versions.index(c2['product_version'])
                        
                        if v2_idx > v1_idx:  # 版本升級
                            upgrades.append({
                                'company_id': cid,
                                'product_line': product_line,
                                'from_version': c1['product_version'],
                                'to_version': c2['product_version'],
                                'v1_date': c1['contract_start_date'],
                                'v2_date': c2['contract_start_date'],
                                'upgrade_days': (c2['contract_start_date'] - c1['contract_start_date']).days,
                                'upgrade_type': '續約升級'
                            })
                    except ValueError:
                        pass  # 版本不在映射表中，跳過
                        
    return pd.DataFrame(upgrades)
```

### 4.3 續約時交叉銷售關聯規則分析

**目標：** 找出「續約時加購」的產品組合模式（例如：續約 ERP 時加購資安產品）

**使用 Apriori 演算法：**
```python
from mlxtend.frequent_patterns import apriori, association_rules

# 建立購物籃矩陣（公司 × 產品）
basket = (
    order_details.groupby(['company_id', 'product_category'])
    .size().unstack(fill_value=0)
)
basket = (basket > 0).astype(int)

# 找出頻繁項目集
frequent_itemsets = apriori(basket, min_support=0.05, use_colnames=True)

# 產生關聯規則
rules = association_rules(frequent_itemsets, metric="confidence", min_threshold=0.3)
rules = rules.sort_values('lift', ascending=False)
```

**輸出範例：**
```
product_A    | product_B    | support | confidence | lift
------------ | ------------ | ------- | ---------- | ----
基礎軟體     | 進階軟體     | 0.12    | 0.60       | 2.5
雲端服務     | 資安產品     | 0.08    | 0.45       | 1.8
```

**解讀：**
- **Support 12%**：12% 的客戶同時購買這兩個產品
- **Confidence 60%**：買了基礎軟體的客戶中，60% 會買進階軟體
- **Lift 2.5**：買基礎軟體後購買進階軟體的機率是整體的 2.5 倍

from pathlib import Path

def parse_event_content(file_path: str, event_name: str) -> dict:
    """解析活動檔案，提取關鍵資訊"""
    
    # 讀取檔案內容
    file = Path(file_path)
    
    if file.suffix.lower() == '.pdf':
        # 使用 Gemini File API

    result = json.loads(response.text.strip().replace("```json", "").replace("```", ""))

### 4.5 沉睡潛客識別與分類 ⭐ 核心功能

**目標：** 從商機記錄中找出有潛力但未成交的客戶，用活動邀約喚醒

**資料來源：**
- `lead_stage_results.csv` — 商機階段記錄
- `contract_summary.csv` — 合約歷史（用於排除已成交客戶）

**識別邏輯：**
```python
def identify_dormant_leads(lead_df, contract_df):
    """識別沉睡潛客"""
    
    # 1. 找出有商機記錄但無合約的公司
    companies_with_leads = set(lead_df['company_id'])
    companies_with_contracts = set(contract_df['company_id'])
    never_converted = companies_with_leads - companies_with_contracts
    
    # 2. 過濾出沉睡潛客（假設 lead_df 有 last_contact_date 欄位）
    dormant_df = lead_df[lead_df['company_id'].isin(never_converted)].copy()
    
    # 3. 計算距離最後接觸天數
    dormant_df['last_contact_date'] = pd.to_datetime(dormant_df['last_contact_date'])
    dormant_df['days_since_contact'] = (pd.Timestamp.now() - dormant_df['last_contact_date']).dt.days
    
    # 4. 過濾條件
    dormant_df = dormant_df[
        (dormant_df['days_since_contact'] >= 90) &  # 至少90天無互動
        (~dormant_df['is_rejected'])                # 未被明確拒絕
    ]
    
    # 5. 分類沉睡類型
    def categorize_dormant(row):
        days = row['days_since_contact']
        stage = row['top_stage']
        
        if days <= 180 and stage in ['C2', 'C1']:
            return '溫沉睡'
        elif days <= 180 and stage in ['E', 'D']:
            return '冷沉睡'
        elif days > 180 and stage in ['C2', 'C1']:
            return '溫沉睡'  # 雖然時間久，但曾經很熱
        else:
            return '冷沉睡'
    
    dormant_df['dormant_category'] = dormant_df.apply(categorize_dormant, axis=1)
    
    # 6. 智能活動推薦（從活動庫中匹配）
    def recommend_event(row, event_library):
        """基於痛點類型與法人類型，從活動庫中推薦最合適的活動"""
        
        pain = row['pain_type']
        cluster = row['cluster_name']
        stage = row['top_stage']
        
        # 過濾：痛點吻合的活動
        matched_events = event_library[
            event_library['pain_types'].str.contains(pain, na=False)
        ]
        
        # 進一步過濾：法人類型吻合
        if not matched_events.empty:
            matched_events = matched_events[
                matched_events['cluster_types'].str.contains(cluster, na=False) |
                matched_events['cluster_types'].isna()  # 或無限制法人類型的活動
            ]
        
        # 根據商機階段選擇活動類型
        if stage in ['C1', 'C2']:
            # 溫沉睡：優先推薦產品發表會、工作坊（深入型）
            preferred_types = ['產品發表會', '工作坊', '解決方案研討會']
        else:
            # 冷沉睡：優先推薦論壇、研討會（廣泛型）
            preferred_types = ['論壇', '研討會', '線上課程']
        
        for ptype in preferred_types:
            filtered = matched_events[matched_events['event_type'] == ptype]
            if not filtered.empty:
                # 取最近的upcoming活動
                upcoming = filtered[filtered['event_status'] == 'upcoming']
                if not upcoming.empty:
                    event = upcoming.iloc[0]
                    return event['event_id'], event['event_name'], f"痛點吻合：{pain}"
        
        # 若無完全吻合，回傳第一個相關活動
        if not matched_events.empty:
            event = matched_events.iloc[0]
            return event['event_id'], event['event_name'], f"主題相關：{pain}"
        
        # 無匹配活動，回傳通用活動
        return None, "產業趨勢論壇", "通用推薦"
    
    # 載入活動庫
    event_library = pd.read_csv("results/phase3/event_library.csv")
    
    dormant_df[['recommended_event_id', 'recommended_event_name', 'match_reason']] = (
        dormant_df.apply(lambda r: pd.Series(recommend_event(r, event_library)), axis=1)
    )
    
    # 7. 計算優先分數（stage權重 + 時間衰減）
    STAGE_SCORE = {'C1': 80, 'C2': 60, 'D': 40, 'E': 20}
    dormant_df['stage_score'] = dormant_df['top_stage'].map(STAGE_SCORE).fillna(10)
    
    # 時間衰減：90-180天=1.0，180-270天=0.8，270-365天=0.6，>365天=0.4
    def time_decay(days):
        if days <= 180:
            return 1.0
        elif days <= 270:
            return 0.8
        elif days <= 365:
            return 0.6
        else:
            return 0.4
    
    dormant_df['time_factor'] = dormant_df['days_since_contact'].apply(time_decay)
    dormant_df['priority_score'] = (dormant_df['stage_score'] * dormant_df['time_factor']).astype(int)
    
    return dormant_df.sort_values('priority_score', ascending=False)
```

    script = response.text.strip()
    
    # 附加報名連結

def batch_generate_scripts(dormant_df, event_library, phase2_labels=None, batch_size=10):
    """批次生成所有沉睡潛客的邀約話術"""

            scripts.append("[話術生成失敗，請手動撰寫]")
        
        # 避免超過 API 速率限制
        if (idx + 1) % batch_size == 0:
            time.sleep(2)
    
    dormant_df['outreach_script'] = scripts
    return dormant_df

def classify_customer(row):
    # 優先處理已成交客戶
    if row["has_contract_history"]:
        # 即將到期（30-90天）
        if pd.notna(row["days_to_expiry"]) and 0 <= row["days_to_expiry"] <= 90:
            return "即將到期"
        # 續約客
        elif row["is_renewal_customer"]:
            # 升級客（有升級歷史的續約客）
            if row["has_upgrade_pattern"]:
                return "升級客"
            else:
                return "續約客"
        # 流失客（合約已過期且未續約）
        elif pd.notna(row["days_to_expiry"]) and row["days_to_expiry"] < 0:
            return "流失客"
        # 新客（有合約但從未續約）
        else:
            return "新客"
    else:
        # 未成交客戶 = 沉睡潛客
        return "沉睡潛客"

### 6.2 業務價值

#### 6.2.1 精準客戶分層
- **升級客（最高優先）**：有升級記錄，高價值，推薦進階方案
- **複購客（高優先）**：持續回購，轉換率高，追加銷售
- **沉睡客（中優先）**：曾經購買但久未回購，喚醒活動
- **新客（一般優先）**：無購買歷史，需求探索階段

#### 6.2.2 預測性行動建議
| 客戶狀態 | 推薦行動 | 觸發條件 |
|---------|---------|---------|
| 回購時機到 | 主動聯繫 | `days_since_last_order ≥ avg_repurchase_days × 0.8` |
| 流失預警 | 緊急跟進 | `churn_risk = high` |
| 升級時機 | 升級方案提供 | 使用基礎版 > 180 天 |
| 交叉銷售 | 產品推薦 | 基於關聯規則（Lift > 1.5） |

#### 6.2.3 產品策略洞察
- **熱門續約路徑**：識別最常見的產品組合順序
- **升級轉換率**：各產品線的升級成功率與週期
- **交叉銷售機會**：高關聯性產品組合（支援套裝銷售設計）

#### 6.2.4 活動行銷自動化 ⭐ 新增
- **智能活動推薦**：基於痛點類型與法人類型，從活動庫自動匹配最合適的活動
- **個人化話術生成**：為每個沉睡潛客生成客製化的邀約話術，節省業務時間
- **活動內容解析**：自動解析活動海報/簡介，提取摘要與標籤，無需手動整理
- **活動效果追蹤**：記錄推薦活動與實際報名/成交的關聯性（未來擴充）
