## 🎯 各階段核心產出

### Phase 1：知道「客戶是誰」
- 8 大屬性：外資/家族/商模/規模...
- 用途：客戶分群基礎

### Phase M：知道「客群特性」
- 7 種聚類：C0 微型內銷 ~ C6 外資貿易
- 用途：精準行銷、資源分配

### Phase 2：知道「客戶痛點」
- L1-L7 深度標籤：痛點/角色/目標/議題/評估/動態/對策
- 用途：銷售話術、產品需求

### Phase 3：知道「做什麼」
- Output 1：229 家需補問的公司
- Output 2：257 個高熱痛點（產品開發）
- Output 3：310 家 B+A 高溫標的（立即跟進）

### Phase 4：知道「有沒有效」⭐ NEW
- 驗證熱圖準確性（相關係數）
- 找出最有價值痛點（轉換率 + ROI）
- 優化商機推進流程（週期分析）
- 準備預測模型（特徵矩陣）

## 💼 各角色使用指南

### 業務主管

**每週使用：**
1. **Phase 3 Output 3**：分配 310 家 B+A 公司給團隊
2. **Phase 4 追蹤表**：檢視跟進進度與成交率

**每月使用：**
1. **Phase 4 B1**：檢視各痛點轉換率，調整話術重點
2. **Phase 4 C1**：分析各階段成交率，找出瓶頸

**每季使用：**
1. **Phase 4 A2**：各聚類產值分析，調整資源配置
2. **重跑 Phase 3-4**：更新熱圖，調整策略

---

### 產品經理

**每季使用（產品規劃）：**
1. **Phase 3 Output 2**：257 個高熱痛點組合
2. **Phase 4 B1**：按 total_revenue 排序，選 Top 10
3. **Phase 4 B3**：按 ROI 排序，評估投資報酬率

**產品開發優先順序：**
```
第一優先：total_revenue > 5,000 萬 AND roi_per_company > 30 萬
第二優先：conversion_rate > 30% AND total_companies > 100
第三優先：heat_score > 0.7（Phase 3 驗證）
```

---

### 業務人員

**每日使用：**
1. **Phase 3 Output 3 追蹤表**：今日應跟進的 3-5 家公司
2. **查看 pain_type**：準備對應話術

**拜訪前準備：**
1. 查詢該公司的：
   - cluster_name：法人類型（了解背景）
   - pain_type：核心痛點（切入點）
   - best_stage：商機階段（推進策略）
   - impact/urgency：影響與緊迫度（談判籌碼）

---

### 資料科學家

**建模準備：**
1. **Phase 4 D1**：feature_matrix.csv
2. 整合以下特徵：
   - 聚類（cluster）
   - 痛點類型（pain_type）
   - 商機階段（top_stage）
   - L1-L7 完整度
   - 歷史成交記錄

**模型目標：**
- 預測成交機率（has_order）
- 推薦系統（最可能成交的 Top N 公司）
- 異常偵測（停滯商機預警）

## 🔄 完整工作流程

### 初始建置（一次性）
```
Day 1-2：運行 Phase 1（如已完成可跳過）
Day 3：  運行 Phase M
Day 4-5：運行 Phase 2（批次處理，28 小時）
Day 6：  運行 Phase 3
Day 7：  運行 Phase 4
Day 8：  檢視結果，制定策略
```

### 每月更新流程
```
Day 1：更新商機階段資料（lead_stage_results.csv）
Day 2：重跑 Phase 3
Day 3：重跑 Phase 4
Day 4：對比上月差異
Day 5：業務會議檢視變化，調整策略
```

### 每季深度優化
```
Week 1：重跑 Phase 2（補充新日報）
Week 2：重跑 Phase 3-4
Week 3：分析趨勢，調整權重
Week 4：產品規劃會議，更新開發路線圖
```
