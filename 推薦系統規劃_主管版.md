# 推薦系統規劃

**版本**：v2.1  
**建立日期**：2026-05-05

---

## 執行摘要

本計畫旨在建立**業務日誌智能推薦系統**，透過 LLM 技術自動化萃取客戶痛點、商機等級與產品推薦標籤，並生成個性化業務話術。系統每日自動處理新增業務日報，T+1 完成標註，大幅提升業務效率。

**關鍵效益**：
- **節省人力成本**：年省 $50,000（1 位專職標註人員）
- **投資報酬率**：25.6 倍（第 1 年）、23.4 倍（5 年累積）
- **業務賦能**：每日自動化標註 685 筆日報，提供 7 大推薦功能（問項/產品/待開發商品/活動/話術/客戶分析）
- **系統成本**：第 1 年 $1,955，5 年累積 $10,698

---

## 1. 系統架構概覽

### 1.1 核心功能

本系統採用多階段 Pipeline 設計，從原始業務日報自動萃取 7 層標籤：

```
原始日報（CRMGY 表）
    ↓
【Phase L】種子庫維護（每週執行）
    ├─ 更新 15 個標籤種子庫（LD001-LD015）
    └─ 重建 Vertex AI Vector Search 索引
    ↓
【Phase 0】增量日報擷取
    └─ 僅處理新增日報，避免重複計算
    ↓
【Phase M】基礎分類（平行執行）
    ├─ 商機等級：E / D / C2 / C1 / B / A
    └─ 法人標籤：外商、家族企業、商業模式、員工人數
    ↓
【Phase 1】深度標籤 L1-L3（KNN 相似度匹配）
    ├─ L1：痛點類型（5 個標籤）
    ├─ L2：客戶類型（5 個標籤）
    └─ L3：解決方案（5 個標籤）
    ↓
【Phase 2】精準標籤 L4-L7（三層分層投票）
    ├─ 分層 1：依商機等級篩選
    ├─ 分層 2：依 L1 痛點分群
    └─ 分層 3：KNN 相似樣本投票（k=30）
    ↓
【Phase 3】合約與客戶行為分析
    ├─ 客戶分類：續約客/即將到期/升級客/新客/流失客/沉睡潛客
    ├─ 活動內容解析（PDF/Word/圖片）
    ├─ 個性化話術生成（150-200 字 + 活動連結）
    └─ 續約路徑分析（產品升級、交叉銷售）
    ↓
【輸出】完整標籤結果 + 業務話術
```

### 1.2 關鍵技術

| 技術組件                  | 用途                          | 供應商           |
|---------------------------|-------------------------------|------------------|
| Gemini Flash LLM          | 商機分類、法人標籤、話術生成  | Google AI        |
| Vertex AI Vector Search   | 深度標籤相似度匹配（L1-L7）   | Google Cloud     |
| SQL Server（DSC_CRM_SP）  | 業務日報與合約資料來源        | 現有基礎設施     |
| Python + FastAPI          | Pipeline 流程控制與 API 服務  | 開源             |
