📁 results/
│
├── 🎯 output1_recommended_questions.csv (229 家)
│   ├─ 用途：找出需要主動補問的商機
│   ├─ 適用：業務主管、業務人員
│   ├─ 時機：每週商機檢視、拜訪準備
│   └─ 關鍵欄位：
│       • company_id (公司 ID)
│       • cluster_name (法人類型：C0~C6)
│       • best_stage (商機階段：C2/C1/B)
│       • missing_layers (缺少的層：L4/L5/L6)
│       • recommended_question (推薦問項)
│
├── 🎨 output2_development_cards.csv (257 組合)
│   ├─ 用途：產品功能開發優先順序
│   ├─ 適用：產品經理、研發主管
│   ├─ 時機：季度產品規劃、ROI 評估
│   └─ 關鍵欄位：
│       • cluster_name (法人類型)
│       • pain_type (痛點類型)
│       • company_count (影響公司數)
│       • ba_ratio (B+A 商機比例)
│       • heat_score (綜合熱度分數)
│
├── 🔥 output3_opportunity_cards.csv (310 家)
│   ├─ 用途：立即跟進的高溫標的（B+A 階段）
│   ├─ 適用：業務團隊
│   ├─ 時機：每日追蹤、週會檢視、月底衝刺
│   └─ 關鍵欄位：
│       • company_id (公司 ID)
│       • cluster_name (法人類型)
│       • best_stage (A=簽約中、B=報價中)
│       • pain_type (核心痛點)
│       • impact (影響範圍)
│       • urgency (緊迫程度)
│
└── 🗺️ pain_heatmap.png
    ├─ 用途：視覺化熱圖（法人類型 × 痛點類型）
    ├─ 適用：高層主管、產品策略
    └─ 看法：紅色區塊 = 高熱需求，優先投資

🎯 三個檔案的使用場景：

業務日常：
    早上 → 開 output3，選 3 家 B+A 公司跟進
    拜訪前 → 查該公司 pain_type，準備話術
    週會 → 檢視 output1，安排補問排程

產品規劃：
    季初 → 看 output2 Top 20 高熱痛點
    月中 → 追蹤 heat_score 變化趨勢
    評估 → 用 ba_ratio 判斷需求成熟度

高層決策：
    戰略會 → 看 pain_heatmap.png 全局
    投資 → 根據 company_count 評估市場規模
    目標 → 用 Output 3 設定季度業績目標
