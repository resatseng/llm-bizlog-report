# Phase 3 輸出檔案使用指南

## 📊 快速總覽

| 輸出檔案 | 用途 | 使用者 | 數量 |
|---------|------|--------|------|
| **Output 1** 推薦業務問項 | 找出需要深入挖掘的商機 | 業務主管 | 229 家 |
| **Output 2** 開發方案卡 | 產品功能開發優先順序 | 產品經理 | 257 組合 |

## 📋 Output 1：推薦業務問項

### 🎯 核心價值
找出「**有痛點、商機溫度夠，但資訊不完整**」的公司 → 業務主動補問可提高成交率

### 📌 篩選邏輯
- ✅ 有 L1 痛點（問題已確認）
- ✅ 商機溫度 ≥ C2（有購買意願）
- ⚠️ 缺少 L4（產業議題）或 L5（評估方案）資訊
- 💡 這些是「半成品商機」，補完資訊即可推進

### 🔍 如何使用

**步驟 1：按法人類型分配**
```
C5 家族外銷 → 110 家  
C1 標準內銷 → 86 家   
C4 品牌驅動 → 21 家   
```
→ 分配給熟悉該類型客戶的業務

**步驟 2：看 missing_layers 欄位**
- 缺 L4 → 問：「目前有哪些外部環境或法規壓力影響決策？」
- 缺 L5 → 問：「在評估哪些解決方案？決策標準是什麼？」
- 缺 L6 → 問：「對現有方向的態度與溫度如何？有近期觸發事件嗎？」

**步驟 3：看 recommended_question 欄位**
→ 系統已自動生成推薦問項，可直接使用

### 💼 業務場景
- **場景 1**：月度商機檢視會議，找出需要主動追蹤的案子
- **場景 2**：業務拜訪前準備，確認要補充哪些資訊
- **場景 3**：商機健檢，避免停滯不前

---

## 🎨 Output 2：開發方案卡

### 🎯 核心價值
找出「**需求明確 + 商機熱 + 規模夠**」的痛點組合 → 指引產品開發優先順序

### 📌 篩選邏輯
- heat_score ≥ 0.5（熱度門檻）
- 熱度公式：`ba_ratio × 0.5 + urgency_high_pct × 0.3 + scale_score × 0.2`
  - `ba_ratio`: B+A 階段公司比例（商機熱度）
  - `urgency_high_pct`: 高緊迫度比例（時間壓力）
  - `scale_score`: 規模分數（市場大小）

### 🔍 如何使用

**步驟 1：看 Top 20 高熱組合**
→ 優先處理 heat_score 最高的痛點

**步驟 2：看 cluster_name + pain_type**
→ 了解「哪種客群」有「什麼痛點」

**步驟 3：看 company_count**
→ 評估市場規模（影響多少家公司）

**步驟 4：看 ba_ratio**
→ 評估商機成熟度（有多少比例是 B/A 階段）

### 📊 關鍵欄位解讀

| 欄位 | 意義 | 如何看 |
|------|------|--------|
| cluster_name | 法人類型 | C5 家族外銷、C1 標準內銷... |
| pain_type | 痛點類型 | 具體的阻礙/問題 |
| company_count | 公司數 | > 10 表示共通需求，值得開發 |
| ba_ratio | B+A 比例 | > 0.3 表示商機熱，容易轉換 |
| heat_score | 綜合熱度 | > 0.7 優先處理，0.5-0.7 次要 |

### 💼 產品場景
- **場景 1**：季度產品規劃會議，決定開發優先順序
- **場景 2**：ROI 評估，選擇投資報酬率最高的功能
- **場景 3**：競品分析，找出尚未被滿足的需求缺口

### 📝 實際案例
```
cluster_name: C5 家族外銷
pain_type: 人員異動問題, 內部流程與預期落差, 關鍵決策者不在
company_count: 1
ba_ratio: 2.0 (200% 表示有重複計數或資料異常)
heat_score: 1.01

→ 解讀：雖然只影響 1 家公司，但該公司處於超高溫狀態（ba_ratio > 1）
→ 行動：個案處理，立即安排解決方案
```

## 🔥 Output 3：機會卡（B+A 高溫標的）

### 🎯 核心價值
「**立即可成交**」的公司清單 → 業務立即跟進，提高短期業績

### 📌 篩選邏輯
- best_stage = A（簽約階段，183 家）
- best_stage = B（報價階段，127 家）
- 這些是商機漏斗最底層，轉換率最高

### 🔍 如何使用

**步驟 1：按 stage_heat 排序**
→ 優先跟進 A 階段（6 分），其次 B 階段（5 分）

**步驟 2：看 cluster_name 分配**
```
C5 家族外銷 → 142 家  
C1 標準內銷 → 114 家  
C4 品牌驅動 → 29 家   
```
→ 分配給熟悉該類型的業務專家

**步驟 3：看 pain_type**
→ 了解客戶的核心痛點，準備對應解決方案

**步驟 4：看 impact / urgency**
→ 評估痛點的嚴重性和時間壓力，調整跟進策略

### 📊 關鍵欄位解讀

| 欄位 | 意義 | 如何看 |
|------|------|--------|
| best_stage | 商機階段 | A=簽約中、B=報價中 |
| stage_heat | 熱度分數 | 6=A、5=B |
| pain_type | 核心痛點 | 拜訪時的切入點 |
| impact | 影響範圍 | 評估案子大小 |
| urgency | 緊迫程度 | 高=立即跟進、低=排程跟進 |

### 💼 業務場景
- **場景 1**：每週業務會議，追蹤 B+A 案件進度
- **場景 2**：月底衝刺，優先收割熱案
- **場景 3**：新人訓練，跟老鳥學習如何推進 B/A 階段案件

### 🎯 跟進策略

| 階段 | 策略 | 動作 |
|------|------|------|
| A (簽約中) | 排除最後障礙 | 確認合約細節、付款條件、上線時程 |
| B (報價中) | 強化價值主張 | 提供 ROI 試算、成功案例、優惠方案 |

## 🔄 三輸出的關係

```
Output 1 (推薦問項)      Output 2 (開發方案卡)      Output 3 (機會卡)
      ↓                        ↓                       ↓
 補充資訊                   產品開發                 立即成交
 推進商機                   長期布局                 短期業績
      ↓                        ↓                       ↓
  C2 → C1 → B              需求驅動開發              B → A → 贏單
  (商機培育)                (產品策略)               (業務執行)
```

## 📈 使用建議

### 對業務主管
1. **每週一**：檢視 Output 3，分配 B+A 案件給業務
2. **每月中**：檢視 Output 1，安排商機補充作業
3. **每季末**：檢視 Output 2，提供給產品團隊

### 對產品經理
1. **每季初**：根據 Output 2 規劃開發路線圖
2. **每月底**：追蹤高熱痛點的需求變化
3. **隨時**：用 heat_score 評估新功能優先順序

### 對業務團隊
1. **每天**：從 Output 3 選 2-3 家跟進
2. **拜訪前**：查看該公司的 pain_type、impact、urgency
3. **會議後**：更新商機階段，讓系統重新計算

---

## 🎓 進階應用

### 1. 動態追蹤
- 每月重跑 Phase 3，觀察熱度變化
- heat_score 上升 → 市場需求增強
- heat_score 下降 → 需求被滿足或轉移

### 2. AB 測試
- 針對 Output 1 推薦問項實驗，比較轉換率
- 驗證補問是否真的提高成交率

### 3. 跨部門協作
- Output 2 → 產品開發
- Output 1 → 業務訓練（如何深度挖掘）
- Output 3 → 業務執行（立即跟進）

## 📞 常見問題

**Q1: 為什麼 Output 3 有些 pain_type 是 NaN？**  
A: 該公司在 B/A 階段，但 Phase 2 深度標籤時未萃取到有效的 L1 痛點（可能是日報內容不足或 LLM 萃取失敗）。仍可跟進，但需現場挖掘痛點。

**Q2: heat_score > 1.0 是異常嗎？**  
A: 是的。ba_ratio > 1 表示資料異常（同一公司被計算多次）。這種案例通常是高價值個案，值得特別關注。

**Q3: Output 2 的 257 個組合太多，如何篩選？**  
A: 
1. 只看 heat_score > 0.7（約 Top 30）
2. 再看 company_count > 5（影響夠多公司）
3. 優先處理你們擅長的 cluster

**Q4: 如何將結果整合進 CRM？**  
A: 建議匯入步驟：
1. Output 3 → 標記為「高溫商機」Tag
2. Output 1 → 建立「待補充」任務
3. 每月更新，動態調整優先順序
