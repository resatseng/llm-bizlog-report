# Phase 4：訂單整合分析

## 目的
整合訂單資料驗證痛需熱圖，計算實際轉換率與產值，建立預測模型基礎。

## 程式
- `L-Phase4訂單整合分析.ipynb` - 訂單資料整合與四大分析

## 輸入資料（五路合流）

| 來源 | 檔案/表 | 說明 |
|------|---------|------|
| Phase 3 | `output1~3.csv` | 推薦問項、開發方案卡、機會卡 |
| Phase 2 | `phase2_deep_labels.jsonl` | L1-L7 深度標籤 |
| Phase M | `company_clusters.csv` | 法人聚類 C0-C6 |
| 商機等級 | `lead_stage_results.csv` | E/D/C2/C1/B/A 階段 |
| **訂單** | `CRMHC` 表 | 實際成交資料 |

## 四大分析

### 📊 分析 A：熱圖驗證

**目的**：驗證 heat_score 是否真的能預測成交

**分析項目：**
1. heat_score vs 實際成交率（相關係數）
2. 各聚類（C0-C6）的實際產值分布
3. Top 20 高熱痛點的成交驗證
4. 模型準確性報告

**輸出：**
- `results/A1_heat_score_validation.csv` - 熱圖驗證報告
- `results/A2_cluster_revenue.csv` - 各聚類產值分析
- `results/A3_validation_chart.png` - 驗證視覺化

### 💰 分析 B：痛點價值分析

**目的**：找出最有價值的痛點類型

**分析項目：**
1. pain_type → 成交率排名
2. pain_type → 平均訂單金額
3. pain_type → 總產值貢獻
4. pain_type ROI 排名（產值/影響公司數）

**輸出：**
- `results/B1_pain_conversion_rate.csv` - 痛點轉換率
- `results/B2_pain_revenue.csv` - 痛點產值分析
- `results/B3_pain_roi_ranking.csv` - 痛點 ROI 排名

### ⏱️ 分析 C：商機週期分析

**目的**：了解商機推進速度與卡點

**分析項目：**
1. 各階段（E→D→C2→C1→B→A）的平均停留天數
2. 從首次接觸到成交的平均週期
3. 商機停滯風險預警（超過平均 2 倍）
4. 快速成交案例分析（< 30 天）

**輸出：**
- `results/C1_stage_duration.csv` - 各階段停留時間
- `results/C2_deal_cycle.csv` - 成交週期分析
- `results/C3_stagnant_warning.csv` - 停滯預警清單

### 🎯 分析 D：預測特徵工程

**目的**：準備建立成交預測模型的特徵

**分析項目：**
1. 整合所有特徵（聚類 + 痛點 + 商機階段 + 時間）
2. 特徵重要性分析
3. 成交機率預測準備（選用）
4. 推薦系統特徵池

**輸出：**
- `results/D1_feature_matrix.csv` - 完整特徵矩陣
- `results/D2_feature_importance.csv` - 特徵重要性
- `results/D3_prediction_ready.csv` - 預測模型資料集

## 關鍵洞察

### 洞察 1：熱圖準確性驗證
```
heat_score ≥ 0.7 的痛點組合 → 實際成交率 XX%
heat_score < 0.5 的痛點組合 → 實際成交率 YY%

驗證 Phase 3 模型是否有效
```

### 洞察 2：最有價值的痛點
```
Top 10 痛點（按總產值）：
1. pain_type_A → 成交率 60% / 平均金額 50 萬 / 總產值 3 億
2. pain_type_B → 成交率 45% / 平均金額 35 萬 / 總產值 2 億
...

指引 Output 2 開發方案卡的優先順序
```

### 洞察 3：商機推進瓶頸
```
E → D：平均 X 天
D → C2：平均 Y 天（最長，是瓶頸）
C2 → C1：平均 Z 天
...

找出需要優化的階段
```

### 洞察 4：成交預測
```
整合 15+ 特徵：
- 聚類（C0-C6）
- 痛點類型（pain_type）
- 商機階段（E/D/C2/C1/B/A）
- L1-L7 完整度
- 停留天數
- 歷史成交記錄
...

準備建立 XGBoost/LightGBM 預測模型
```

## 使用建議

### 對業務主管
1. **每月看分析 A**：驗證熱圖是否持續準確
2. **每季看分析 B**：調整產品開發優先順序
3. **每週看分析 C**：找出停滯商機，主動推進

### 對產品經理
1. **用分析 B** 的 ROI 排名決定開發順序
2. **用分析 A** 驗證需求是否真實
3. **用分析 D** 準備個人化推薦功能

### 對資料科學家
1. **用分析 D** 的特徵矩陣建立預測模型
2. **用分析 C** 優化商機評分邏輯
3. **用分析 A** 調整 heat_score 權重

## 後續應用

### 應用 1：動態熱圖
每月更新訂單資料 → 重算熱圖 → 調整權重

### 應用 2：智能推薦
根據成交歷史，推薦最有可能成交的公司給業務

### 應用 3：預警系統
商機停滯超過平均 2 倍 → 自動發送預警

### 應用 4：績效評估
業務人員跟進高 heat_score 案件的成交率 → KPI
